WHAT I BUILT / SHIPPED

üåê Flutter Web Versions

I shipped Flutter web versions of all my apps using WASM (WebAssembly). This was actually a good learning experience because Flutter web with WASM is still relatively new. Now you can use Ritualz, Expensif, and Tuteee right in the browser without downloading anything.

üìä Analytics Dashboards

I built analytics dashboards for all my apps. This is so important because I need to understand how users are actually using the apps. Without data you're just guessing. Now I can see daily active users, what platforms they're on, how often they're completing actions, etc.

üí∞ Monetisation on All Apps

All three apps now have monetisation in place. The model I've settled on is: free to use with ads, or subscribe to pro to remove ads and unlock premium features. I've also been experimenting with rewarded ads - watch an ad to get a reward. This feels like a nice balance where users can choose.

üîó Social Share Feature (Ritualz)

üìù Feedback Board

WHAT I LEARNED

üåê Flutter Web with WASM

Flutter web has gotten a lot better with WASM support. The performance is noticeably smoother than the old JavaScript renderer. It's not perfect but it's good enough to ship web versions of mobile apps now.

üí∞ Monetisation Strategies

I've been thinking a lot about monetisation. The "watch an ad for a reward" model feels good because it gives users a choice. Either pay for pro, or watch an ad to get the benefit temporarily. It doesn't feel as intrusive as forced ads. I'll keep experimenting with this.

üîî Supabase Edge Functions - Push Notifications

Learned how to set up push notifications using Supabase Edge Functions. This is important for habit apps like Ritualz because reminders help users stay consistent. Edge functions are pretty powerful for running server-side code without managing your own backend.
